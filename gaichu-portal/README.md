# 外注先ポータル（さくらのレンタルサーバ）— Phase 2 / 閲覧片方向

契約済み案件の **地図・図面・写真・納期／納骨日／彫刻箇所** を、外注先に
パスワード付きで閲覧してもらう仕組みです。データは内部アプリ（`index_b.html`）の
**「外注先へ公開」** ボタンから、さくらへ自動でPush（アップロード）されます。

- マスタは常に **SharePoint**。さくらは「配布用の箱」（閲覧専用）。
- 外注先は **URL＋パスワード（Basic認証）** だけ。Microsoftアカウントは不要。
- いまは **片方向（閲覧のみ）**。外注先からの写真UP・コメント・進捗入力は Phase 3。

## ファイル構成

```
www/
  robots.txt                     ← 検索避け（/gaichu/ を Disallow）
  gaichu/
    yamada/                      ← 外注先ごとのフォルダー（Basic認証）
      index.php                  ← ポータル本体（このリポジトリ）
      upload.php                 ← 庄司→さくら 受信（合言葉必須／このリポジトリ）
      submit.php                 ← 外注→さくら 報告受信（Basic認証内・合言葉不要／このリポジトリ）
      .htaccess                  ← パスワード（Basic認証）＋ noindex
      .htsecret                  ← アップロード合言葉（1行）※upload.php用
      .user.ini                  ← PHP上限（任意・写真が大きい場合）※user.ini-sample.txt参照
      cases/                     ← アプリが自動生成（手で触らない）
        k-123/  case.json  地図/  図面/  写真/
        n-45/   case.json  資料/  報告/  comments.json
        c-45/   case.json  地図/  写真/
```

`case.json` はアプリが書き出します（お寺名・家名・分類・納期／納骨日＋時間・状態・彫刻箇所・更新日など）。
`cases/` 配下はアプリ管理領域なので、FTPで手作業する必要はありません。

## セットアップ（さくら・初回のみ）

1. **FTP接続**し、公開フォルダー（通常 `/home/アカウント名/www/`）に `gaichu/yamada/` を作成。
2. `index.php` と `upload.php` をそのフォルダーにアップロード。
3. **合言葉ファイル `.htsecret` を作成**（アップロードの認証に使用）。
   - 長いランダム文字列を1行だけ書く。例（ターミナル）: `echo -n '長いランダム文字列' > .htsecret`
   - `.ht` で始まるファイルは Apache が既定で配信拒否するため、外から中身は見えません。
   - この文字列は **内部アプリ側にも1回だけ登録**します（端末に保存・公開ソースには載せません）。
4. `robots.txt` を **`/www/` 直下**にアップロード（既にあれば `Disallow: /gaichu/` を追記）。
5. **パスワード（Basic認証）をかける**（どちらか）
   - かんたん：さくらの **コントロールパネル →「アクセス制限」** で `yamada/` にユーザー名・パスワードを設定。
   - 手動：`htaccess-sample.txt` を元に `.htaccess` を作成（`AuthUserFile` は絶対パス）。
6. 外注先に **URL（例 `https://〇〇.sakura.ne.jp/gaichu/yamada/`）とパスワード** を渡す。

### `.htsecret` と `upload.php` の許可設定
- `upload.php` は、内部アプリの配信元（`https://111rbubu-art.github.io`）からのPOSTのみ CORS 許可します。
  別ドメインで運用する場合は `upload.php` 冒頭の `$ALLOW_ORIGINS` に追記してください。
- 受け付ける拡張子は `upload.php` の `$ALLOW_EXT` にホワイトリストしています（実行系は拒否）。

## 使い方（運用）

1. 内部アプリで対象案件（工事関連／納骨リスト）の詳細を開く。
2. **「外注先へ公開」** を押す → その案件の項目とファイル（ドキュメントライブラリ＋添付）が
   さくらへPushされてポータルにIN。内容を更新したら再度押せば **まるごと差し替え**。
3. **「公開を外す」** を押すと、その案件フォルダーがさくらから削除されます。
4. ポータルは **納期（納骨日／彫刻完了予定）の早い順**、工事／納骨／彫刻タブ・未完／完了で表示。
   画像はタップで拡大、PDF等はリンクで開きます。

## セキュリティ

- 公開リンク（誰でも）ではなく **パスワード必須**。ファイルもフォルダーごと保護。
- `noindex` + `robots.txt` で検索避け（※noindexは「検索に出さない」だけで、パスワードが本体の防御）。
- アップロードは **合言葉（.htsecret）** と **CORS制限** で保護。
- 外注先との関係が終わったら、`yamada/` のパスワードを変える／フォルダーを消すだけで遮断。
- 「見せた案件の中身」を相手が保存・印刷することは仕様上防げません（作業に必要なため）。
  守れるのは「未公開・他社の案件を見られないこと」です。

## Phase 3（外注先からの報告）— 実装済み

外注先がポータル上で **コメント** と **写真** を投稿でき、庄司石材は内部アプリで閲覧・削除できます。
保存先はさくらのみ（SharePointには入れません）。詳しくは `PHASE3_SPEC.md`。

- **設置**：`submit.php` を各外注先フォルダー（`upload.php` と同じ場所）にアップロードするだけ。
  - `submit.php` は **Basic認証の中**に置き、認証は外さない（＝認証済み外注のみ投稿可）。
  - `upload.php` はアプリからのPOST用に認証除外が必要だが、`submit.php` は除外しない。
- **保存**：写真＝`cases/<id>/報告/<名前>_<YYYYMMDD>_<連番>.<ext>`、コメント＝`cases/<id>/comments.json`。
- **投稿者名**：`GAICHU_NAME`（環境変数）を既定に、無ければ初回にポータルで入力（端末に記憶）。
- **写真が大きい場合**：`user.ini-sample.txt` を参考に `.user.ini` で上限を上げる。
- **庄司側**：内部アプリの「🌐 外注公開」管理画面に「📮 外注からの報告」欄（写真サムネ＋🗑削除・コメント一覧・件数表示）。
