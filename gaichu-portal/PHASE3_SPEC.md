# 外注ポータル Phase 3 仕様（外注先からの報告：コメント＋写真アップ）

外注先が**ポータル上で**コメントと写真を投稿でき、庄司石材は内部アプリで閲覧・削除できる。
保存先は**さくらのみ**（SharePointへは取り込まない）。通知はなし（アプリで見分けられれば十分）。

## 1. データの置き場所（さくら）
```
cases/<caseId>/
  case.json              ← 庄司がPushした案件情報（既存）
  地図/ 図面/ 写真/ 資料/  ← 庄司が共有したファイル（既存）
  報告/                   ← ★外注先がアップした写真（新規・区別用）
    内田_20260715_01.jpg
    内田_20260715_02.jpg
  comments.json          ← ★外注先のコメント（新規）
```
- 外注アップ分は必ず `報告/` 配下 → 庄司共有分とフォルダで区別。
- `comments.json` 形式：`[{ "name":"内田", "at":"2026-07-15 14:32", "text":"…" }, …]`

## 2. ファイル名ルール（写真）
- **`<投稿者名>_<YYYYMMDD>_<2桁連番>.<拡張子>`**（例 `内田_20260715_01.jpg`）
- 連番＝その案件 `報告/` 内の「同じ名前＋日付」プレフィックスの数＋1（当日通し）。
- 投稿者名：index.php が持つ表示名（環境変数 `GAICHU_NAME` 等）を既定に。
  無ければポータルのフォームで一度入力→ブラウザに記憶（localStorage）。

## 3. さくら側：`submit.php`（新規）
- **Basic認証の中**に置くので、投稿できるのは認証済みの外注先のみ。
- **同一オリジン**（ポータルと同じフォルダ）＝ CORS不要・合言葉不要。
  ※ 庄司→さくらの `upload.php`（合言葉必須）とは**別ファイルで用途分離**。
- アクション（POST, multipart）：
  - `comment` … `{case, name, text}` → `comments.json` に1件追記
  - `upload`  … `{case, name, file}` → `報告/` に保存（上記ファイル名規則、1件ずつ）
  - `list`    … `{case}` → `報告/` の写真一覧 ＋ `comments.json` を返す（ポータル描画用）
  - `del`     … `{case, path}` → 自分の投稿を削除（任意機能）
- 保護：
  - 拡張子ホワイトリスト：`jpg jpeg png gif webp heic heif pdf`
  - 1ファイルサイズ上限：約30MB（`.user.ini` で `upload_max_filesize`/`post_max_size`）
  - `case` は `[A-Za-z0-9_-]{1,64}` のみ／パストラバーサル無効化
  - 書き込みは `報告/` と `comments.json` に限定

## 4. ポータル（index.php）UI追加
- 各カードに「**報告**」セクション：
  - 📷 写真を追加（複数選択・カメラ可）→ `submit.php upload` を1枚ずつ
  - 💬 コメント入力＋送信 → `submit.php comment`
  - 投稿済みの**写真サムネ**と**コメント一覧**を表示（`submit.php list`）
  - 送信後はその場で再取得して反映
  - （任意）自分の投稿に🗑削除
- ついでに Phase 3 と同時に：**🔄 リロードボタン**を上部に追加（TODO項目）。

## 5. 内部アプリ（index_b.html）側
- 公開管理画面に「**外注からの報告**」欄を追加：
  - `報告/` の写真をサムネ表示（既存 `getfile`）＋ **🗑削除**（既存 `delfile` がそのまま使える）
  - コメント一覧を表示（`upload.php` に `comments` アクションを追加して `comments.json` を返す）
  - **見分け用インジケータ**：報告写真件数＋コメント件数を表示（例「外注報告 写真3・コメント2」）。
    既読管理は簡易（件数のみ）。通知は行わない。
- 外注アップ分は `報告/` 配下なので、庄司共有分と自動で区別可能。

## 6. 実装順序
1. さくら：`submit.php` 追加＋`.user.ini`（サイズ上限）
2. ポータル：`index.php` に報告UI（コメント・写真アップ・一覧・リロード）
3. 内部アプリ：管理画面に「外注からの報告」欄（閲覧・削除・件数表示）＋`upload.php` に `comments` 追加

## 7. 保存先の注記
- 報告はすべて**さくらに保存**（SharePointには入れない）。
- 将来SharePointへ取り込みたくなったら、その時に別途「取り込み」機能を追加する。
