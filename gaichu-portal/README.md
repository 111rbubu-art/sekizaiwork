# 外注先ポータル（さくらのレンタルサーバ）— Phase 1

契約済み案件の **地図・図面・写真・納期** を、外注先ごとにパスワード付きで閲覧してもらう仕組みです。
Phase 1 は「さくらに置くだけ」で動きます（既存アプリには影響しません）。

## 全体像

- ファイルは **さくらのレンタルサーバ**（300GB・FTP・PHP）に置く
- 外注先ごとにフォルダーを分け、**フォルダー単位でパスワード（Basic認証）**
- 検索エンジンには出さない（noindex / robots）
- 外注先は「自分のURL＋パスワード」で、**自分に共有された案件だけ**閲覧

```
www/
  robots.txt                ← このリポジトリの robots.txt を /www 直下へ
  gaichu/
    a-sha/                  ← A社用（パスワードA）
      index.php             ← このリポジトリの index.php
      .htaccess             ← htaccess-sample.txt を元に作成（or さくら管理画面で設定）
      山田家墓石工事/
        info.txt
        地図/  map1.jpg …
        図面/  zumen.pdf …
        写真/  p1.jpg …
      佐藤家納骨/
        …
    b-sha/                  ← B社用（パスワードB）
      index.php
      .htaccess
      …
```

## セットアップ手順（さくら）

1. **FTP接続**し、公開フォルダー（通常 `/home/アカウント名/www/`）に `gaichu/a-sha/` と `gaichu/b-sha/` を作成。
2. 各社フォルダーに **`index.php`** をアップロード。
3. `robots.txt` を **`/www/` 直下**にアップロード（既にある場合は `Disallow: /gaichu/` の行を追記）。
4. **パスワードをかける**（どちらか）
   - かんたん：さくらの**コントロールパネル →「アクセス制限」**で各フォルダーにユーザー名・パスワードを設定。
   - 手動：`htaccess-sample.txt` を参考に `.htaccess` を作成し、`.htpasswd` を用意（`AuthUserFile` は絶対パス）。
5. 各社に **URL（例 `https://〇〇.sakura.ne.jp/gaichu/a-sha/`）とパスワード**を渡す。

## 案件を追加する（運用）

1. その外注先フォルダー（例 `a-sha/`）の中に **案件名のフォルダー**を作る。
2. 中に `地図` `図面` `写真` などのサブフォルダーを作り、ファイルを入れる（サブフォルダー名がそのまま見出しになります。直下に置いたファイルは「資料」にまとまります）。
3. 任意で **`info.txt`** を置くと、案件名・納期・メモを表示できます：
   ```
   案件名: 山田家 墓石工事
   納期: 2026/08/10
   メモ: 図面は最新版のみ参照してください
   ```
4. 画面は **納期の早い順**に自動で並びます。画像はサムネイル表示、PDF等はリンク表示。

## セキュリティ

- 公開リンク（誰でも）ではなく **パスワード必須**。ファイルもフォルダーごと保護されます。
- `noindex` + `robots.txt` で検索避け。
- 外注先との関係が終わったら、その社フォルダーのパスワードを変える／フォルダーを消すだけで遮断できます。
- 「見せた案件の中身」を相手が保存・印刷することは仕様上防げません（作業に必要なため）。守れるのは「他社・未共有の案件を見られないこと」です。

## Phase 2（自動化・任意）

いまは「案件フォルダーへFTPでファイルを入れる」手動運用です。
ご希望があれば次段で、**内部アプリ（index_b.html）に「外注先へ共有」ボタン**を付け、
契約案件のSharePoint上の図面・写真を **さくらへ自動アップロード＋一覧生成**するところまで作れます
（さくらにアップロード受け取り用のPHPを1つ追加。アップロード用の合言葉は端末ごとに保存し、公開ソースには載せません）。
